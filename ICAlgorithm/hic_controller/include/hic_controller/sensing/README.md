# `sensing/` 使用文档

## 这个目录是做什么的

`sensing/` 主要负责“交互力/扭矩传感器相关观测”。  
当前核心头文件是：

- `hic_interaction_force_observer.h`

## 它和 `state/` 有什么区别

这两个目录很容易混淆，但职责不同：

### `state/`

更偏“机器人本体状态”，例如：

- 位置
- 速度
- 加速度
- 电机电流
- 估计力矩

### `sensing/`

更偏“交互力感知”，例如：

- 原始扭矩传感器输入
- 标定后的关节扭矩传感器输出
- 外部关节扰动力矩
- 滤波和故障状态

## 当前核心类

### `HicInteractionForceObserver`

主要负责：

- 扭矩传感器配置管理
- 从文件加载传感器配置
- 原始通道数据输入
- 通道到关节的标定和映射
- 滤波
- 饱和/故障检查
- 关节外力矩输入和读取

## 一个非常容易踩坑的点

下面两种量不能默认当成一回事：

- `calibratedJointTorqueSensor`
  这是“传感器标定后的关节扭矩值”
- `jointExternalTorque`
  这是“外界扰动力矩估计值或外部直接输入值”

两者物理来源和语义都不一样。

## 当前版本和零力示教的关系

当前 ZeroForceMode 第一版没有依赖复杂外力估计。  
也就是说：

- 不用 `externalWrench`
- 不把 `motorCurrent` 当作末端六维力
- 不走复杂 force-guided 逻辑

所以 `sensing/` 当前更多是为后续扩展和传感器链路保留能力，  
而不是零力示教主链路的核心。

## 什么时候你应该重点看这个目录

- 扭矩传感器标定不对
- 硬件通道和关节映射有问题
- 想做关节外力估计
- 后续想扩展更复杂的人机交互力控

## 一句总结

`sensing/` 是“交互力相关输入”的地方，  
它不是机器人本体状态观察器，也不是主控制器。
